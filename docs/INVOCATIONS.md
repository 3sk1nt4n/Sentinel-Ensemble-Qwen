# Sentinel Qwen Ensemble - Invocation Contracts

coordinator.py calls the model **5 times** per run - the 4 numbered invocations
below **plus the Step-13AA self-correction finalize**. Odd invocations (1, 3)
use MCP tools; even invocations (2, 4) are text-only reasoning. Each invocation
is **stateless** - the model never sees its own previous output; the conductor is
the memory.

> **Current implementation note.** The model runs on **Qwen (Alibaba Cloud
> DashScope) by default**, selected by env through the provider seam
> (`src/sift_sentinel/llm_provider.py`); the identical pipeline also runs on the
> **Anthropic API** as a fallback. Calls go **directly to the provider's API**,
> not the Claude Code CLI. The
> `invoke_model()` wrapper further down illustrates the stateless
> timeout/fallback **contract** (max-turns, timeouts, fallbacks, state-via-files)
> that the API path preserves. The **Step-13AA finalize** (a single AI pass)
> superseded the per-finding Step-12 self-correction loop shown in the table's
> last row. For the live component flow and the 5-call diagram, see
> [`ARCHITECTURE.md`](../ARCHITECTURE.md).

|---|---|---|---|---|---|---|---|

| **2** | Steps 8-9 | Analyze + write findings | 1 (text) | 120s | ALL tool outputs, filtered by prepare_prompt() | Structured findings (Pydantic JSON) | Retry with Golden Path outputs only |
| **3** | Step 11 | Investigation threads (ReAct, self-correction layer 1) | 5-7 (MCP) | 90s | Validated findings + raw tool outputs | Thread requests: {artifact, tools[], hypothesis} | Skip threads, use Inv 2 findings |
| **13AA** | Step 13 | Finalize self-correction (re-judge every non-terminal finding; self-correction layer 2) | 1 (text) | - | Non-terminal findings + evidence cross-refs | Verdicts: confirmed / needs-review / FP / unresolved | Keep prior dispositions |

## Timing Budget

```
Mount + SHA256 + SSDT         30s   (deterministic)
Mandatory tools               60s   (Volatility is slow)
INVOCATION 1                  60s   ← timeout
Run selected tools            60s   (depends on count)
Build reference set            5s   (deterministic)
INVOCATION 2                 120s   ← timeout (longest)
Validate findings              5s   (deterministic)
INVOCATION 3                  90s   ← timeout
Run thread tools + validate   30s   (deterministic)
Self-correction loop          90s   ← shared window
Confidence calibration         5s   (deterministic)
INVOCATION 4                  90s   ← timeout
Validate report                5s   (deterministic)
SHA256 again                   5s   (deterministic)
                             ─────
                            ~655s worst case (10.9 min)
                            ~360s likely case (6 min)
```

## State Transfer

Each invocation communicates via files, not memory:

```
/tmp/sift-sentinel-run-<run-id>/     # one scratch dir per run (illustrative)
├── evidence/              # mounted read-only
├── tool_outputs/          # JSON per tool (vol_pstree.json, etc.)
├── reference_set.json     # built by reference_set.py
├── inv1_prompt.md         # generated by coordinator.py
├── inv1_response.json     # AI response (provider API - DashScope/Qwen by default)
├── inv2_prompt.md
├── inv2_response.json
├── findings_validated.json
├── inv3_prompt.md
├── inv3_response.json
├── findings_final.json    # after self-correction + confidence
├── inv4_prompt.md
├── report.md              # final output
├── audit_trail.jsonl      # append-only log
└── sha256_pre.json / sha256_post.json
```

## invoke_model() - the stateless call contract

The call goes **directly to the provider's API** (Qwen on Alibaba Cloud
DashScope by default; Anthropic via the same seam - never the Claude Code CLI).
The shape below is the original Anthropic-era illustration of the stateless
timeout/fallback contract the conductor enforces around every invocation - each
call is independent, state lives in files:

```python
def invoke_model(prompt_path: str, timeout: int, max_turns: int, fallback_fn):
    # model = qwen3.7-max (HEAVY) or qwen-plus (LIGHT), resolved by the provider
    # seam (llm_provider.py) from env - live calls go to Alibaba Cloud DashScope
    # (OpenAI-compatible). max_turns bounds tool-use rounds for the MCP
    # invocations (1, 3); the text-only invocations (2, 4) use max_turns = 1.
    try:
        resp = llm_complete(                     # llm_provider.py seam
            prompt=open(prompt_path).read(),
            timeout=timeout,
        )
        return parse_and_validate(strip_markdown_fences(resp.text))
    except (LLMTimeout, LLMAPIError) as e:
        log("TIMEOUT_OR_API_ERROR", prompt_path, str(e))
        return fallback_fn()
    except (json.JSONDecodeError, ValidationError) as e:
        log("INVALID_RESPONSE", prompt_path, str(e))
        return fallback_fn()
```

*(Illustrative - the Anthropic fallback provider preserves this identical
contract; only the seam's backend changes.)*

## Key Helpers

- **prepare_prompt()**: Filters tool outputs within token budget before Invocation 2. Prevents context window blowout.
- **strip_markdown_fences()**: Removes ```json``` wrappers from the model's output. 5 lines of defensive parsing.
- **ThreadPoolExecutor**: Parallel tool execution for Volatility plugins (I/O-bound). Can cut 60s to 15-20s.
